## 修复 NOT NULL constraint failed: customers_new.name (13:22)

### 问题
用户报告：`migrateRemoveEmailColumn()` 迁移失败，提示 `NOT NULL constraint failed: customers_new.name`

### 原因分析
`INSERT INTO customers_new SELECT` 语句直接复制 customers 表的 name 字段到 customers_new 表，但：
1. 旧 customers 表可能存在 name 值为 NULL 的记录
2. customers_new 表的 name 列定义为 NOT NULL
3. SQLite 不允许向 NOT NULL 列插入 NULL 值

### 修复方案
修改 `migrateRemoveEmailColumn()` 中的 INSERT SELECT 语句：
- 使用 `COALESCE(name, '')` 替换直接引用 `name`
- 当 name 为 NULL 时，使用空字符串作为默认值

## 修复并发迁移和数据库损坏问题 (13:54)

### 问题
用户报告：`table customers_new already exists` 和 `no such table: customers_new` 交替出现

### 原因分析
1. React StrictMode 会 double invoke effects，导致两个迁移同时执行
2. 第一个迁移正在执行时，第二个也开始执行
3. 相互干扰导致表被删除但新表未创建完成

## 修复到期状态筛选"已到期"不正确问题 (14:15)

### 问题
用户报告：筛选"已到期"不正确显示，筛选"未到期"正确。

### 原因分析
原逻辑使用 `MAX(end_date) < today` 判断"已到期"：
- 如果客户**没有服务费记录**，`MAX(end_date)` 返回 `NULL`
- `NULL < today` 在 SQL 中结果是 UNKNOWN（既不是 true 也不是 false）
- 这些客户被过滤掉，不显示在任何筛选结果中

### 修复方案
改用 `NOT EXISTS` 判断"已到期"：
- "已到期"：`NOT EXISTS (SELECT 1 FROM service_fee_records WHERE ... AND start_date <= today AND end_date >= today)`
- "未到期"：`EXISTS (SELECT 1 FROM service_fee_records WHERE ... AND start_date <= today AND end_date >= today)`
- 两者逻辑对称，都是检查当前是否有有效服务费记录

- `inventory-management/src/services/database/CustomerService.ts`
  - `getAllCustomers()` 方法中的 feeStatusFilter 筛选逻辑
